Інструкція з експлуатації: Професійний емулятор OBD-II (ESP32-S3)
1. Опис пристрою
Цей емулятор є програмно-апаратним комплексом на базі мікроконтролера ESP32-S3, призначеним для повної імітації роботи до 4-х електронних блоків керування (ECU) автомобіля:

ECM (Двигун) — адреса 0x7E0 / 0x18DAF110
TCM (Трансмісія) — адреса 0x7E1 / 0x18DAF118
ABS (Гальмівна система) — адреса 0x7E2 / 0x18DAF128
SRS (Подушки безпеки) — адреса 0x7E3 / 0x18DAF158
Пристрій підтримує протоколи ISO 15765-4 CAN (11-біт та 29-біт) зі швидкостями 250/500 kbps та стеком UDS (ISO 14229).

2. Технічні характеристики
Процесор: Dual-core ESP32-S3.
CAN-інтерфейс: Вбудований контролер TWAI з підтримкою ISO-TP сегментації.
Дисплей: TFT ST7735 (128x160) для моніторингу статусу.
Керування: Веб-інтерфейс через Wi-Fi (HTTP + WebSocket).
Пам'ять: Збереження налаштувань в NVS та можливість експорту в JSON.
3. Підключення
3.1. Апаратна частина
Живлення: 5V через USB-C або зовнішнє джерело.
CAN шина: Підключіть лінію CAN-H та CAN-L до діагностичного роз'єму або сканера.
Важливо: Якщо лінія довга, переконайтеся у наявності термінуючого резистора 120 Ом між CAN-H та CAN-L.
Піни (типово): TX — GPIO 4, RX — GPIO 5.
3.2. Мережеве підключення
Пристрій може працювати у двох режимах:

Access Point (AP): SSID: OBD-II-Emulator-A, пароль: 123456789. Адреса: 192.168.4.1.
Station (STA): Підключення до вашої локальної мережі Wi-Fi (налаштовується у вкладці "Network").
4. Функціональні можливості
4.1. Режими OBD-II (J1979)
Mode 01: Передача поточних даних (PIDs). Підтримується понад 60 стандартних параметрів.
Mode 02: Freeze Frame (стоп-кадр), що фіксується в момент появи першої помилки.
Mode 03/07/0A: Читання підтверджених, очікуваних та постійних кодів помилок (DTC).
Mode 04: Очищення помилок та скидання адаптацій.
Mode 06: Результати тестів самодіагностики (TIDs).
Mode 09: Інформація про автомобіль (VIN, Calibration ID, CVN).
4.2. Спеціальні симуляції
У вкладці General можна активувати:

Dynamic RPM: Оберти змінюються за синусоїдою (імітація роботи двигуна).
Misfire Simulation: Автоматична генерація помилки P0300 при високих обертах.
Lean Mixture Sim: Генерація P0171 при зниженні тиску палива.
4.3. Робота з DTC (Помилками)
Емулятор має інтелектуальну систему розподілу кодів:

Коди P та U записуються в блок ECM.
Коди C дублюються в ECM та ABS.
Коди B дублюються в ECM та SRS. Це гарантує, що будь-який стандартний OBD-сканер "побачить" всі помилки через основний протокол, а дилерські сканери зможуть знайти їх у відповідних блоках.
5. Впорскування несправностей (Fault Injection)
Цей розділ призначений для тестування стійкості вашого ПЗ або сканера:

Response Delay: Штучна затримка відповіді (0-1000 мс).
CAN Error Rate: Відсоток пакетів, які будуть випадково проігноровані.
ISO-TP Faults: Помилкові номери послідовності (Sequence Numbers), переповнення STmin, або відправка некоректних Flow Control кадрів.
6. Керування даними
Apply Changes: Тимчасово оновлює параметри в пам'яті.
Save to Device (NVS): Записує поточний стан у енергонезалежну пам'ять (параметри відновляться після перезавантаження).
Config JSON: Дозволяє скачати повний профіль налаштувань на ПК або завантажити готовий файл конфігурації.
7. Моніторинг
TFT Дисплей: Відображає поточну IP-адресу, вибраний режим CAN та активні помилки.
CAN Monitor: У веб-інтерфейсі можна спостерігати за сирим трафіком (RX/TX кадри) в реальному часі.
Live Chart: Графічне відображення RPM, Speed, Temp та MAF.
8. Усунення несправностей
Немає зв'язку зі сканером: Перевірте вкладку "General" — чи співпадає Bitrate (500k/250k) та тип ID (11/29 bit).
Веб-сторінка не відкривається: Перевірте, чи ви підключені до Wi-Fi мережі емулятора та чи правильно вказано IP (за замовчуванням 192.168.4.1).
Помилки дублюються в списку: Веб-інтерфейс автоматично фільтрує дублікати для зручності, але сканер може бачити їх від різних блоків — це нормальна поведінка для мульти-ЕБУ системи.

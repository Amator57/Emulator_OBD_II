# Інструкція з експлуатації

## Інструкція з експлуатації: Емулятор OBD-II (ESP32-S3)

### 1. Опис пристрою
Цей пристрій призначений для імітації роботи електронного блоку керування (ЕБУ) автомобіля за протоколом **OBD-II** (`ISO 15765-4 CAN`). Емулятор дозволяє тестувати діагностичні сканери (наприклад, `ELM327`), мобільні застосунки (`Torque`, `Car Scanner`) та власне програмне забезпечення без необхідності підключення до реального автомобіля.

### 2. Технічні характеристики
- Мікроконтролер: `ESP32-S3` (Dual-core, **Wi-Fi**/Bluetooth).
- Протокол зв'язку: **CAN Bus** (`TWAI`) зі швидкістю `500 kbps` (стандарт для сучасних авто).
- Інтерфейси керування:
    - **Веб-інтерфейс** (через **Wi-Fi**).
    - **TFT-дисплей** для відображення поточного стану.
    - `USB-CDC` для моніторингу та прошивки.
- Кількість підтримуваних кодів помилок (DTC): понад `120` (стандарт `P0xxx`).

### 3. Підключення

#### 3.1. Апаратне підключення
Для роботи з **CAN-шиною** необхідно використовувати трансивер (наприклад, `SN65HVD230` або `TJA1050`).

- **CAN High / CAN Low**: Підключіть до відповідних пінів вашого **OBD-II** сканера або діагностичного роз'єму.
- Живлення: Через `USB-порт` розробника (`5V`) або через зовнішній стабілізатор `12V` -> `5V`, якщо підключення йде безпосередньо до бортової мережі.
- **Піни ESP32-S3**: TX — `GPIO 4`, RX — `GPIO 5` (типово для проекту).

#### 3.2. Налаштування Wi-Fi
Після увімкнення пристрій створює точку доступу **Wi-Fi** (або підключається до існуючої мережі, залежно від налаштувань).
Перейдіть за `IP-адресою` пристрою (за замовчуванням зазвичай `192.168.4.1`) у браузері.

### 4. Функціональні можливості

#### 4.1. Емуляція параметрів (Mode 01)
Емулятор передає поточні дані, такі як:

- Оберти двигуна (`RPM`).
- Швидкість автомобіля (`Vehicle Speed`).
- Температура охолоджуючої рідини.
- Напруга бортової мережі.

#### 4.2. Робота з помилками (Mode 03)
Емулятор може імітувати наявність несправностей. У пам'яті закладено таблицю стандартних кодів, включаючи:

- Помилки системи паливоподачі (`P0001-P0004`).
- Помилки датчиків положення розподільчого валу (`P0010-P0015`).
- Помилки датчиків витрати повітря (`P0100-P0104`).
- Помилки запалювання та пропусків іскри (`P0300-P0308`).

#### 4.3. Очищення помилок (Mode 04)
Пристрій підтримує команду очищення кодів несправностей (`Clear DTCs`), що дозволяє перевірити логіку роботи діагностичного софту.

### 5. Використання веб-інтерфейсу
Через веб-панель ви можете:

- Вибирати активні коди помилок (DTC), які буде бачити сканер.
- Змінювати значення `PID` (наприклад, «підняти» оберти двигуна повзунком).
- Спостерігати за логами `CAN-трафіку` в реальному часі.

### 6. Усунення несправностей
- Сканер не бачить пристрій: Перевірте підключення `CAN High/Low` та наявність термінуючого резистора `120 Ом` (якщо лінія довга).
- Не відкривається веб-сторінка: Перевірте, чи підключені ви до **Wi-Fi** мережі емулятора.
- Помилка ініціалізації `CAN`: Переконайтеся, що піни `TWAI` в коді відповідають фізичному підключенню до трансивера.

## Поради щодо розширення (Code Quality Review)
З огляду на ваш код у файлі `ESP-32.c`, рекомендую:

- Використання `PROGMEM`: Оскільки таблиця `DTC` досить велика (`~120` записів), варто зберігати її у `флеш-пам'яті`, щоб не перевантажувати `RAM`.
- Динамічна відповідь: У функції `sendDTC()` зараз зашито статичні коди. Рекомендую переписати її так, щоб вона брала коди з масиву `active_dtcs`, який заповнюється через **веб-інтерфейс**.
